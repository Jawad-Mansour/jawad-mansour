## Overview

AI engineer who builds the unglamorous parts as carefully as the clever ones. My work lives where machine learning meets real software — large language models, retrieval systems, agentic and multi-agent setups, and the MLOps plumbing that keeps all of it standing up in production.

I came into AI from a full-stack and backend background, and that shaped how I work: I care about drift detection and clean architecture with the same weight I give to model accuracy — because a model nobody can deploy or trust is just an expensive notebook. I move comfortably between research and engineering: benchmarking models, then shipping them behind authenticated APIs with CI/CD, observability, and guardrails.
